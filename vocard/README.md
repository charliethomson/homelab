# Vocard

Self-hosted Discord music bot. Vocard plays audio from YouTube, Spotify, SoundCloud, and others via a Lavalink audio server. A web dashboard provides remote control. MongoDB persists playlists and user data.

## Services

| Service | Port | Purpose |
|---|---|---|
| vocard | — | Discord bot |
| vocard-dashboard | 8003 | Web UI / remote control |
| vocard-db | — | MongoDB — playlists and user data |
| lavalink | — | Audio server (internal) |
| yt-cipher | — | YouTube cipher solver for Lavalink |
| spotify-tokener | — | Anonymous Spotify token provider for Lavalink |

All services are on an isolated `vocard` Docker network. Only the dashboard is exposed to the host.

## Directory structure

```
vocard/
├── docker-compose.yml
├── run.sh                      # renders config templates and wraps docker compose
├── .env                        # secrets — gitignored
├── .env.example
├── settings.json               # bot config template (${VAR} placeholders)
├── dashboard/
│   └── settings.json           # dashboard config template
├── lavalink/
│   ├── application.yml         # lavalink config template
│   ├── plugins/                # downloaded JARs — gitignored
│   └── logs/                   # gitignored
├── mongodb_data/               # MongoDB data directory — gitignored
├── patches/                    # bind-mount overrides; path mirrors container filesystem
│   └── app/cogs/task.py        # extended idle timeout (30 min) before disconnect
└── live/                       # rendered configs generated by run.sh — gitignored
```

`run.sh` substitutes all `${VAR}` placeholders in the three template files into `live/` before starting the stack. Docker mounts from `live/`.

## Initial setup

### 1. Environment file

```bash
cp .env.example .env
```

Fill in all values in `.env`:

| Variable | Where to get it |
|---|---|
| `DISCORD_TOKEN` | Discord Developer Portal → your app → Bot → Token |
| `DISCORD_CLIENT_ID` | Developer Portal → your app → OAuth2 → Client ID |
| `DISCORD_CLIENT_SECRET` | Developer Portal → your app → OAuth2 → Client Secret |
| `GENIUS_TOKEN` | genius.com/api-clients (optional, for lyrics) |
| `LAVALINK_PASSWORD` | anything — only used internally |
| `MONGO_USERNAME` | anything |
| `MONGO_PASSWORD` | anything |
| `DASHBOARD_IPC_PASSWORD` | anything — shared between bot and dashboard |
| `DASHBOARD_SECRET_KEY` | random string: `openssl rand -hex 32` |
| `SPOTIFY_CLIENT_ID` | developer.spotify.com/dashboard |
| `SPOTIFY_CLIENT_SECRET` | developer.spotify.com/dashboard |

### 2. Start

```bash
./run.sh up -d
```

### 3. Invite the bot

Go to Discord Developer Portal → your app → OAuth2 → URL Generator. Select scopes `bot` and `applications.commands`, grant the permissions the bot needs, and use the generated URL to invite it to your server.

## Patches

Files under `patches/` are bind-mounted over the corresponding paths inside the `vocard` container, mirroring the container filesystem exactly. For example, `patches/app/cogs/task.py` is mounted at `/app/cogs/task.py`.

**`patches/app/cogs/task.py`** — overrides the upstream task cog to extend the idle disconnect timeout to 30 minutes (upstream disconnects almost immediately). The rest of the logic is unchanged.

To patch another file, find its path inside the container and create it at the same relative path under `patches/`, then add a volume entry to the `vocard` service in `docker-compose.yml`.

## Day-to-day usage

```bash
./run.sh up -d
./run.sh stop
./run.sh logs -f vocard
./run.sh pull && ./run.sh up -d
```

## Troubleshooting

**Bot is online but won't play** — check that Lavalink is healthy: `./run.sh ps`. If it's not, check logs: `./run.sh logs lavalink`. Lavalink downloads its plugins on first start; give it a minute.

**Spotify tracks not resolving** — confirm `SPOTIFY_CLIENT_ID` and `SPOTIFY_CLIENT_SECRET` are correct in `.env` and that the Spotify app on developer.spotify.com is active. Also check `./run.sh ps` to confirm `spotify-tokener` is healthy.

**`spotify-tokener` stuck unhealthy** — this container occasionally fails its healthcheck. Restart it: `./run.sh restart spotify-tokener`.

**Dashboard unreachable** — confirm `vocard-dashboard` is healthy (`./run.sh ps`) and that the `redirect_url` in `dashboard/settings.json` matches the OAuth redirect URI configured in the Discord Developer Portal.

**Config changes not taking effect** — the live configs in `live/` are only regenerated when `run.sh` is invoked. After editing `.env` or a template, re-run `./run.sh up -d` to regenerate and recreate affected containers.
